# VanLife

A collection of small web apps and resources for our RV
(**2022 Winnebago Travato 59K**), served from one GitHub Pages site.

The published site lives in **`docs/`** — GitHub Pages serves that folder as the site root.

## Apps
| App | Source | Live URL | Description |
|-----|--------|----------|-------------|
| Landing page | [`docs/index.html`](docs/index.html) | `…/VanLife/` | Home page listing all apps. |
| How to Use Our Van | [`docs/user-guide/`](docs/user-guide/) | `…/VanLife/user-guide/` | Mobile/tablet how-to guide (installable PWA, offline) for anyone using the van. |

_(Add more apps as new folders under `docs/` and register them in `apps.json`.)_

## Publishing
One command builds every app and regenerates the landing page in `docs/`:

```
npm run publish        # or: node scripts/publish.js
```

`scripts/publish.js` reads **`apps.json`** (the app registry), runs each app's build
step (e.g. Markdown → JSON), and rewrites `docs/index.html` to list every app. Then
commit and push — GitHub Pages serves `/docs`.

### Add a new app
1. Create its folder under `docs/<slug>/` (a self-contained static site / PWA).
2. Add an entry to `apps.json`:
   ```json
   { "slug": "<slug>", "title": "…", "description": "…", "build": "scripts/build.js", "icon": "<slug>/icons/icon-192.png" }
   ```
   (Omit `"build"` if the app has no build step.)
3. Run `npm run publish`.

## Repo layout
```
.
├── README.md
├── apps.json                 # app registry (used by scripts/publish.js)
├── scripts/publish.js        # builds all apps + regenerates the landing page
└── docs/                     # ← GitHub Pages root
    ├── .nojekyll             # serve files as-is (no Jekyll build)
    ├── index.html            # landing page (generated by publish)
    └── user-guide/           # the "How to Use Our Van" PWA (self-contained)
        └── README.md         # how to edit content, add photos, deploy
```

## Local development (Vite)
A Vite dev server previews the site locally at the same subpath GitHub Pages uses.

```
npm install        # one time
npm run dev        # → http://localhost:5173/VanLife/  (landing page)
                   #   http://localhost:5173/VanLife/user-guide/  (the guide)
npm run content    # rebuild data/content.json after editing guide-content.md
```

`vite.config.mjs` sets `root: docs` and `base: /VanLife/` so local URLs match production.
`npm run build` (optional) outputs a static copy to `dist/`; the site normally ships straight
from `docs/`, so a build step isn't required.

## Enable GitHub Pages (one time)
**Settings → Pages → Build and deployment → Source: Deploy from a branch →
branch `main`, folder `/docs`.** Save.

The site is then live at `https://<username>.github.io/VanLife/`, and each app at
`https://<username>.github.io/VanLife/<app>/`. See each app's own README for details.
